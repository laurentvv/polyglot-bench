## 🔧 Project Configuration

### Compilation Script

The project uses `compile_cpp.bat` for automated compilation:

```batch
@echo off
setlocal enabledelayedexpansion

set "source_file=%1"
set "output_file=%2"

if "%source_file%"=="" (
    echo Usage: compile_cpp.bat source_file [output_file]
    exit /b 1
)

if "%output_file%"=="" (
    for %%f in ("%source_file%") do set "output_file=%%~nf.exe"
)

call "C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvars64.bat" >nul 2>&1

cl /O2 /EHsc "%source_file%" /Fe:"%output_file%" >nul 2>&1

if !errorlevel! equ 0 (
    echo Compilation successful: %output_file%
    exit /b 0
) else (
    echo Compilation failed for %source_file%
    exit /b 1
)
```

### Benchmark Configuration

In `bench.config.json`:


## 🔍 Troubleshooting

### Common Issues

**1. "cl is not recognized"**
```cmd
# Solution: Run vcvars64.bat first
call "C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvars64.bat"
cl
```

**2. "Cannot open include file"**
- Verify Visual Studio installation includes Windows SDK
- Check include paths in c_cpp_properties.json

**3. "Compilation failed"**
```cmd
# Debug compilation manually
cd tests\algorithms\fibonacci
cl /O2 /EHsc fibonacci.cpp /Fe:fibonacci.exe
```

**4. VSCode IntelliSense not working**
- Reload VSCode window (Ctrl+Shift+P → "Developer: Reload Window")
- Check C++ extension is enabled
- Verify c_cpp_properties.json paths

### Performance Optimization

- `/O2` - Maximum optimization for speed
- `/EHsc` - Exception handling model
- Modern C++17 standard features
